Anthropic 连放两个大招：
昨天接管你的电脑，今天甩掉你的审批

48 小时内，两个重磅功能上线。Anthropic 正在有意识地把 AI 从「听指挥干活」变成「自己判断着干活」。

3 月 24 日，Claude Desktop 上线 Computer Use，让 AI 直接操控鼠标和键盘，能干你坐在桌前能干的所有事。

3 月 25 日，Claude Code 推出 Auto Mode，AI 写代码时不再每个操作都来问你「行不行」，自己判断安全性，自己执行。

放在一起看，这不只是两个功能更新——而是 Anthropic 在同一个方向上踩了两脚油门。

Computer Use：从文件助手到桌面操控

在 Computer Use 之前，Claude Desktop 的 Cowork 模式已经能帮你读写文件、做表格、写报告。但它有一个明显的边界：只能动文件系统里的东西。让它查个网页、在某个桌面 App 里点几下、或者打开 Excel 填个数据，它就只能摊手。

Computer Use 打破了这个边界。Claude 现在可以截取屏幕来「看」当前状态，然后通过控制鼠标和键盘操作任何应用程序。技术策略很务实：优先走 API 集成（比如已接入的 Slack、Calendar），只有没有 API 可用时才退回到屏幕操控这种「兜底方案」。

👁️
截屏感知
Claude 截取当前桌面状态，理解「现在屏幕上有什么」。
🖱️
鼠标键盘控制
直接操控光标、输入文字，任意 App 均可操作。
📱
Dispatch 远程派单
手机发一句话，Claude 在你电脑上默默把活做完，进会议室时已经完成。
🔌
API 优先策略
有 API 就走 API，没有才用屏幕操控，减少无意义的截图开销。

最有想象力的场景是配合手机端的 Dispatch 功能。你在外面用手机发「帮我把那个 PPT 导出成 PDF，附到下午两点的会议邀请里」，Claude 就在你电脑上默默完成了所有操作。等你走到会议室，活已经做好了。

三层安全防护
1
手动开启确认：功能默认关闭，用户主动启用
2
逐个 App 授权：每个应用单独赋权，不打包给
3
权限粒度设置：每个 App 可选「完全控制」或「仅查看」，操作期间其他窗口隐藏

⚠️ 目前仅限 macOS，Pro 和 Max 用户可用，标注为 Research Preview。

Auto Mode：用 AI 审查 AI

用过 Claude Code 的人都知道默认模式的痛点：每个文件写入、每条 shell 命令都要手动确认，跑一个大任务可能要按上百次「允许」。之前的「解放方案」是 --dangerously-skip-permissions，但连名字都在警告你：你在冒险。

自动模式的做法是引入一个独立的分类器模型（固定使用 Sonnet 4.6），在每个操作执行前做语义级的安全审查。这个分类器不是简单的规则匹配，它分析用户的意图和 Claude 想执行的操作，判断有没有超出任务范围，有没有被恶意内容「带偏」的迹象。

安全的操作自动放行，危险的操作自动拦截。被拦截后 Claude 不会卡住，而是换一种方式继续干活。

架构亮点

工具执行的结果不会发送给分类器。即使 Claude 读取了一个精心构造的恶意文件，文件内容也无法影响分类器的判断。分类器只看「用户说了什么」和「Claude 想做什么」，不看「Claude 看到了什么」。这从架构上切断了 prompt injection 影响安全决策的路径。

三层决策链
1
预设规则：已知安全/危险模式直接分类
2
自动放行：通过规则的操作直接执行，不打扰用户
3
分类器审查：模糊地带交给 Sonnet 4.6 做语义判断，连续误判则自动降级回手动模式

⚠️ 目前仅限 Team 计划用户，Enterprise 和 API 即将跟进。

放在一起看：同一个方向，两脚油门

两个更新背后有一个清晰的共性：都在扩大 Claude 的自主行动范围，同时用分层的安全机制来兜底。

维度 Computer Use Auto Mode
扩大的是 操作空间（整个桌面） 决策空间（自主判断执行）
限制机制 手动开启 → App 授权 → 权限粒度 预设规则 → 放行 → 分类器审查
安全思路 多层权限收敛 独立模型互不信任
当前限制 仅 macOS，Pro/Max 仅 Team，Enterprise 待来

两者的安全思路惊人地一致：不信任单一环节，用多层检查来降低风险。

Anthropic 显然在有意识地推进一个更大的叙事：AI 不再只是一个等你输入指令的工具，而是一个能在你不在场时独立工作的代理。Computer Use 让它有了「手脚」，Auto Mode 让它有了「判断力」，Dispatch 让你可以远程「派任务」。三者组合起来，一个能自主行动的 AI 工作伙伴的轮廓已经相当清晰了。

核心要点速览
  • Computer Use 让 Claude 接管整个桌面，不只是文件系统
  • Auto Mode 引入独立分类器，在不问用户的情况下自主决策
  • 分类器只看「意图和操作」，不看「执行结果」，从架构上防 prompt injection
  • 两个功能的安全设计思路一致：多层检查，不信任单一环节
  • 配合 Dispatch，你在外面用手机发任务，回来活就做完了
  • Anthropic 的方向很清晰：从「听指挥」到「自主代理」
